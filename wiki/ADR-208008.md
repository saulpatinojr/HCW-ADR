## Context

The PACS Group Azure estate and the appex workload IaC repository require a consistent, repeatable mechanism for producing and maintaining Azure architecture diagrams that can be:

- Version-controlled alongside Terraform code in GitHub
- Reviewed as part of pull requests with visual diffs
- Shared with clients as portable PDF or PNG exports without licensing constraints
- Kept current across dev, tst, and prd environments as infrastructure evolves

Specific gaps driving this decision:

- No standard diagramming tool is defined for the engagement — team members use ad-hoc tools (PowerPoint, Visio, Lucidchart, whiteboard photos) producing inconsistent artifacts
- Architecture diagrams are not co-located with the IaC code they represent, causing diagrams to drift out of sync with deployed resources
- Existing diagrams do not use the Microsoft Azure icon set, making them difficult to map to Azure Portal resources during reviews
- No icon syntax standard exists — previous attempts to use draw.io produced diagrams where icons rendered as grey boxes on other machines due to use of the stencil format (`shape=mxgraph.azure2.*`) instead of the image library format (`image=img/lib/azure2/...svg`)
- Architecture documentation is not correlated with CAF naming conventions, WAF pillars, or compliance frameworks (PCI-DSS, NIST), making it difficult to use diagrams as governance artifacts

Without a governed diagramming standard, architecture diagrams cannot serve as auditable infrastructure documentation or be included in client deliverables with confidence.

---

## Decision

We have decided to adopt **draw.io** (diagrams.net) as the standard Azure architecture diagramming tool for all PACS Group engagements and the appex workload IaC repository, with the following specific conventions:

**File format:** All diagrams are stored as `.drawio` XML files committed to the same Git repository as the IaC code they document. The file is treated as source code — changes require a pull request, diagram updates accompany the Terraform changes they describe.

**Icon format:** All Azure icons use the image library format exclusively:
```
image=img/lib/azure2/<category>/<Icon_Name>.svg
```
The stencil format (`shape=mxgraph.azure2.*`) is prohibited. The image library format is built into draw.io and does not require external stencil packs, ensuring diagrams render correctly on any machine with draw.io installed.

**Deployment targets:** Three deployment modes are supported and interchangeable:
- draw.io Desktop (Windows / macOS / Linux) — primary for diagram authoring
- draw.io Web (https://app.diagrams.net) — secondary for browser-based access
- draw.io VS Code Extension (hediet.vscode-drawio) — recommended for IaC development workflows

**Naming and colour conventions:** All diagrams follow the CAF naming convention (`<type>-<workload>-<env>-<region>`) and the colour coding scheme defined in the appex IaC repository (`NAMING-TAGGING.md`):

| Element | Fill | Border |
|---------|------|--------|
| Connectivity Subscription | #EBF5FB | #1e40af |
| Workload Subscription | #F0FDF4 | #16a34a |
| Resource Group | #DBEAFE | #3b82f6 |
| VNet / Active Subnet | #d1fae5 | #059669 |
| Reserved Subnet | #f0fdf4 | #86efac (dashed) |

**Required annotations:** Every architecture diagram must include:
- Title block with workload name, environment, and date
- CAF resource names on all icons
- Traffic flow arrows with protocol and port labels
- Legend with colour coding and arrow type reference
- Footnotes for ARM-required fixed names and placeholder values

**Change management:** Architecture diagrams are updated in the same pull request as the Terraform resources they represent. Merging Terraform changes without an accompanying diagram update is prohibited.

---

## Drivers

- Establish a single, version-controllable architecture diagramming standard for the PACS Group engagement portfolio
- Ensure Azure diagrams use the official Microsoft icon set and render consistently across all team machines and client environments
- Enable pull request-based diagram review as part of the IaC change process
- Produce portable client deliverables (PDF, PNG) without per-seat licensing costs
- Align architecture documentation with CAF naming, WAF pillars, and PCI-DSS/NIST compliance frameworks
- Eliminate the icon rendering failure caused by use of the stencil format on machines without the stencil pack installed

## Alternatives Considered

| Alternative | Pros | Cons | Reason Rejected |
| --- | --- | --- | --- |
| Microsoft Visio | Industry-standard; full Azure stencil library; VSDX format | Paid licence ($15–$28/user/month); binary .vsdx format is not diffable in Git; not portable for client handoff without Visio licence | Licence cost and non-diffable format disqualify it for IaC-co-located documentation |
| Lucidchart | Good Azure icon set; collaborative editing | Freemium (limited shapes on free tier); proprietary format; requires account; no offline mode | Licence dependency and format lock-in; not suitable for offline or client-delivered artifacts |
| Mermaid.js | Text-based; native GitHub rendering; no install required | No Azure icon support; diagrams are flowcharts only; cannot represent spatial network topology (subnets, peering) | Lack of Azure icons and inability to represent VNet topology disqualifies it for architecture diagrams |
| Excalidraw | Free; hand-drawn aesthetic; VS Code extension available | No Azure icon library; hand-drawn style not appropriate for client deliverables | No Azure icon support |
| PowerPoint / Keynote | Familiar; no new tool required | Not diffable; icons are inconsistent across users; not version-controllable; cannot be embedded in Confluence or GitHub | Not appropriate for IaC-co-located architecture documentation |
| Azure Architecture Center templates | Officially aligned | Visio or PowerPoint format only; not draw.io compatible; requires Visio for editing | Format dependency on Visio |

## Consequences

### Positive

- Architecture diagrams live alongside Terraform code in Git — diagrams are always current with the deployed infrastructure
- GitHub renders `.drawio` files as visual previews in the file browser, enabling reviewers to see diagram changes without installing draw.io
- The image library icon format (`image=img/lib/azure2/...svg`) renders correctly on any machine with draw.io installed — no stencil pack dependency
- draw.io is free with no per-seat licence cost, eliminating procurement friction for client engagements
- Diagrams serve as both visual architecture documentation and CAF naming reference — resource names are embedded in the icons
- PDF and PNG export enables direct inclusion in client FinOps and governance deliverables

### Negative / Trade-offs

- draw.io does not natively support auto-generating diagrams from Azure Resource Graph queries — diagrams must be maintained manually
- The `.drawio` XML diff in GitHub is textual (coordinate changes, style string changes) — the visual diff requires the draw.io GitHub app or manual comparison
- Team members unfamiliar with draw.io require onboarding — the comprehensive guide (`drawio-azure-complete-guide.pdf`) addresses this
- Diagrams showing swimlane-nested containers (subscriptions → resource groups → VNets → subnets) become large — A1 landscape page size is required for complex architectures

### Risks

- If team members revert to using the stencil format (`shape=mxgraph.azure2.*`), icon rendering failures will recur on machines without the stencil pack. The image library format requirement is enforced in this ADR and documented in the appex README.
- Icon SVG paths are case-sensitive — `Key_Vaults.svg` works, `key_vaults.svg` does not. The complete icon path index in `drawio-azure-complete-guide.pdf` Section 14 is the authoritative reference.
- ARM-required subnet names (`GatewaySubnet`, `AzureFirewallSubnet`, `AzureBastionSubnet`) must be shown with their exact names in diagrams to reflect Azure enforcement — diagrams that show incorrect names create client confusion.

## Implementation Notes

- **Primary file:** `appex-iac/appex-architecture.drawio` — committed to the appex IaC repository
- **Icon format rule:** `image=img/lib/azure2/<category>/<Icon_Name>.svg` — no exceptions
- **Prohibited format:** `shape=mxgraph.azure2.*` — stencil format, renders as grey box without stencil pack
- **Page size:** A1 Landscape (2200×1700px) for complex multi-subscription diagrams
- **Default font:** Helvetica 7–8pt for icon labels; Helvetica Bold 8pt for container titles
- **Export settings:** PDF with Fit Page enabled; PNG at 200% scale minimum for client deliverables
- **GitHub:** `.drawio` files render as visual previews natively. Add `*.drawio linguist-language=XML` to `.gitattributes`
- **VS Code:** Install extension `hediet.vscode-drawio` — any `.drawio` file opens in the built-in editor
- **Reference guide:** `drawio-azure-complete-guide.pdf` — covers installation, configuration, icon syntax, patterns, shortcuts, and troubleshooting
