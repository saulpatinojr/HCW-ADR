## Context

The PACS Group Azure estate spans 21 subscriptions across a single tenant. There is no consistent mechanism for collecting, normalizing, and presenting cost and resource data to stakeholders in a repeatable, auditable format.

Specific gaps driving this decision:

- Cost data lives in Azure Cost Management but is not correlated with subscription ownership, management group hierarchy, or tagging compliance in a single view
- Shared services (networking, security, monitoring) are not allocated to consuming workloads — costs appear as unattributed overhead
- Tag compliance is not measured or reported systematically, making chargeback and showback models unreliable
- Defender for Cloud plan coverage is not inventoried across subscriptions
- There is no exception register for subscriptions or resources that fall outside policy

Without a governed collection and reporting process, cost accountability cannot be established and FinOps maturity cannot progress.

---

## Decision

We have decided to implement a PowerShell-based FinOps workbook generation pipeline (`Populate_Workbook.ps1`) that produces a structured Excel workbook covering all 21 subscriptions in the tenant.

The workbook is the authoritative cost visibility artifact for the engagement and contains the following governed sheets:

- **Billing Validation** — 3-month cost summary per service per subscription, sourced from Azure Cost Management
- **Subscription Inventory** — full subscription roster with state, management group assignment, resource group count, and owner fields
- **MG Hierarchy** — management group tree with subscription membership
- **Tag Assessment** — per-subscription tagging compliance rate, RAG status, and missing mandatory tag identification
- **Defender for Cloud Plans** — plan name, pricing tier, and free trial status per subscription
- **Shared Services** — shared infrastructure inventory (networking, security, monitoring) for cost allocation modeling
- **Direct Allocation Model** — per-subscription cost center and business unit mapping for direct chargeback
- **Shared Allocation Formulas** — proportional split methodology for shared service cost distribution
- **Exception Register** — subscriptions or resources with documented policy exceptions, risk level, and treatment

The script authenticates via Windows Web Account Manager (WAM) through `Connect-AzAccount` and requires the following Az PowerShell modules: `Az.Accounts`, `Az.Resources`, `Az.Billing`, `Az.Security`, `Az.ResourceGraph`, and `ImportExcel`.

Mandatory tags evaluated for compliance are defined in the script configuration block and must be kept in sync with the tagging policy defined in ADR-208005.

---

## Drivers

- Establish a repeatable, tenant-wide cost visibility artifact for the PACS Group FinOps engagement
- Enable direct and shared cost allocation modeling across 21 subscriptions
- Measure and track tagging compliance as a prerequisite for chargeback
- Inventory Defender for Cloud coverage as a security governance input
- Create an auditable exception register for out-of-policy resources

---

## Alternatives Considered

| Alternative | Pros | Cons | Reason Rejected |
| --- | --- | --- | --- |
| Azure Cost Management exports to Storage Account | Native, automated | No correlation with hierarchy, tags, or Defender data; requires separate tooling to normalize | Does not produce a unified governance artifact |
| Power BI FinOps dashboard | Rich visualization | Requires Premium capacity or per-user license; not portable for client handoff; no offline artifact | License dependency and delivery friction |
| Manual Excel workbook populated by hand | No tooling required | Not repeatable; error-prone across 21 subscriptions; cannot be version-controlled | Not scalable or auditable |
| Third-party FinOps platform (e.g., Apptio, CloudHealth) | Full-featured | Significant cost; procurement lead time; overkill for engagement scope | Cost and timeline not justified |

## Consequences

### Positive

- Stakeholders have a single, portable Excel artifact covering all 21 subscriptions
- Cost allocation modeling can begin immediately from the Shared Services and Direct Allocation sheets
- Tagging compliance is quantified per subscription with a RAG status, enabling prioritization
- The workbook is regenerable on demand — data is always current to the run date
- Exception register creates an auditable trail for policy deviations

### Negative / Trade-offs

- The workbook is a point-in-time snapshot; it does not update automatically between runs
- Billing data accuracy depends on Cost Management API availability and subscription access scope
- Disabled or warned subscriptions will still appear in the inventory with zero cost rows, requiring manual review
- The script requires interactive WAM authentication; unattended scheduling is not supported without service principal adaptation

### Risks

- If mandatory tags in the script configuration drift from ADR-208005, compliance scoring will be incorrect
- Subscriptions added to the tenant after a run will not appear until the script is re-executed
- Shared services allocation methodology in the workbook requires human review before use in financial reporting

---

## Implementation Notes

- Script: `Populate_Workbook.ps1`
- Output: `OUTPUT_Workbook_Working.xlsx` (written to the directory from which the script is executed)
- Authentication: WAM via `Connect-AzAccount`; tenant is hard-filtered to `93c1680b-e1a5-4495-a6ba-9d1d4ea523b0`
- Mandatory tag list is defined in the `$MandatoryTags` configuration block at the top of the script
- Run with `-Force` to overwrite all existing worksheet data; omit to skip sheets that already contain data
- Run with `-FreshLogin` to force re-authentication when switching tenant contexts
- The script emits a Run Log sheet on every execution recording per-section completion status and warnings
